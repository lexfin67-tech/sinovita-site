# SinoVita Ingredients — статичный сайт

Готовый модуль: 4 страницы + общие стили. Чистый HTML/CSS, без сборки и зависимостей.

## Файлы
- `index.html` — главная
- `products.html` — каталог (extracts / minerals / capsules / packaging)
- `about.html` — о компании
- `contact.html` — контакты + форма заявки
- `style.css` — общие стили (один файл на все страницы)
- `illustrations.js` — встроенные SVG-иллюстрации сырья (порошки, капсулы, флакон и т.д.). Подключён на всех страницах, ничего настраивать не нужно.

> **Важно:** при загрузке на хостинг грузи `illustrations.js` вместе с остальными файлами, иначе иллюстрации категорий не отрисуются (карточки не сломаются, но останутся пустыми панелями).

1. **Email** — найди по всем файлам `sales@sinovita-ingredients.com`, замени на свой.
2. **Телефон** — `+86 21 5888 XXXX` → свой номер.
3. **WeChat / Telegram / WhatsApp** — в `contact.html`:
   - `https://wa.me/8621XXXXXXXX` → свой WhatsApp (формат: wa.me/КОДНОМЕР без +)
   - `https://t.me/sinovita` → свой Telegram
   - `SinoVita_Sales` → свой WeChat ID
4. **Форма заявки** (`contact.html`) — строка `action="https://formspree.io/f/YOUR_FORM_ID"`:
   - Зарегистрируйся на https://formspree.io (бесплатно)
   - Создай форму, получишь ID вида `xayzqwer`
   - Вставь его вместо `YOUR_FORM_ID`
   - Заявки будут падать тебе на email
5. **Футер** — текст «Demonstration site...» поменяй на свой. **ICP-номер убран намеренно:** он нужен только для хостинга внутри материкового Китая и проверяется по базе MIIT. На GitHub Pages (хостинг вне Китая) выдуманный ICP — это проверяемое несоответствие, а не плюс к доверию. Если когда-нибудь будешь хоститься в Китае и получишь настоящий — впишешь.

## Публикация на GitHub Pages (бесплатно)

1. Зарегистрируйся на https://github.com
2. Создай новый репозиторий (Repositories → New), например `sinovita-site`, отметь **Public**.
3. На странице репозитория: **Add file → Upload files** → перетащи `index.html`, `products.html`, `about.html`, `contact.html`, `style.css`, `illustrations.js` → **Commit changes**.
4. **Settings → Pages → Branch:** выбери `main` / `(root)` → **Save**.
5. Через 1–2 минуты сайт будет доступен по адресу `https://ТВОЙ_ЛОГИН.github.io/sinovita-site/`.

### Свой домен
1. Купи домен (reg.ru / namecheap / porkbun) — ~10–15 €/год. Для китайского образа лучше `.com`.
2. В **Settings → Pages → Custom domain** впиши домен → Save.
3. У регистратора домена в DNS добавь записи:
   - 4 × `A`-записи на `185.199.108.153`, `185.199.109.153`, `185.199.110.153`, `185.199.111.153`
   - либо `CNAME` на `ТВОЙ_ЛОГИН.github.io`
4. Включи **Enforce HTTPS** (галочка появится после привязки домена).

SSL-сертификат GitHub выдаёт автоматически.

## Фото вместо иллюстраций (опционально)

Сайт уже укомплектован **встроенными SVG-иллюстрациями** сырья (порошок с листьями,
колба с кристаллами, молекула, капсулы, флакон, OEM-бирка) и линейными иконками в галерее.
Они лицензионно чистые (оригинальная векторная графика), ничего не подгружают извне и не ломаются.

Если захочешь заменить иллюстрации на **реальные фотографии**:

**Где брать бесплатные коммерческие фото:**
- Unsplash — https://unsplash.com (free for commercial use, без атрибуции; избегай кадров с узнаваемыми брендами и лицами).
- Pexels — https://pexels.com (аналогично).
- Pixabay — https://pixabay.com (лучше скачать к себе, а не хотлинкать).

**Как заменить:** в карточке категории (`index.html`) убери `data-ill="..."` у `<div class="cat-img">`
и задай ему `style="background-image:url('ССЫЛКА');background-size:cover"`. Поисковые запросы:
`supplement powder`, `capsules macro`, `herbal extract powder`, `warehouse pallets`.

> Бесплатные стоки не дают юридической индемнификации. Для серьёзного коммерческого сайта
> надёжнее свои съёмки или платный сток с покрытием.
